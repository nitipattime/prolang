คำสั่งวนซ้ำ (Iterative Statements)

การออกแบบต้องพิจารณา

  1. การควบคุมการวนซ้ำนั้นจะกำหนดอย่างไร ?
    • Counter-Controlled Loops ใชัตัวนับควบคุมการวนลูปหรือ
    • Logically-Controlled Loops ใช้นิพจน์ควบคุมการวนลูป
    • Iteration Based on Data Structures ใช้โครงสร้างข้อมูล ควบคุมการวนลูป
  2. การควบคุมการวนซ้ำในลูปจะกำหนดอย่างไร ?
    • User-Located Loop Control Mechanisms ผู้ใช้งานกำหนดการควบคุมการวนซ้ำเอง เช่น คำสั่ง , หรือ คำสั่ง (ปัจจุบันไม่พบในภาษาทั่วไป)

Mechanisms คำสั่งควบคุมจะสามารถใช้งานได้กับลูปเดียว เช่น break; คำสั่งควบคุมจะสามารถใช้งานได้มากกว่าลูปเดียว เช่น ภาษา PHP มี break ระบุจำนวน; และ continue ระบุจำนวน; 

while (i < 0)
            while (j <0) {
                 break 2;
             }
        }

ใน PHP ตัวเลขที่กำหนดคือการนับลูป โดยลูปปัจจุบันเป็น 1 และนับลูปจากลูปปัจจุบันไปหา Ancestor unit เช่น ลูปในเงื่อนไข (i<0) เป็น Ancestor unit ของ (j <0) ซึ่งถือว่าเป็นลูปที่ 2 ตามคำสั่ง break 2;

ภาษา Java มี break และ continue ที่ระบุด้วย label

class ContinueWithLabelDemo {
            public static void main(String[] args) {
        
                String searchMe = "Look for a substring in me";
                String substring = "sub";
                boolean foundIt = false;
        
                int max = searchMe.length() -
                          substring.length();
        
            test:
                for (int i = 0; i <= max; i++) {
                    int n = substring.length();
                    int j = i;
                    int k = 0;
                    while (n-- != 0) {
                        if (searchMe.charAt(j++) != substring.charAt(k++)) {
                            continue test;
                        }
                    }
                    foundIt = true;
                        break test;
                }
                System.out.println(foundIt ? "Found it" : "Didn't find it");
            }
        }

ข้อเสีย การควบคุมในลักษณะนี้จะทำให้เกิดความสับสนในการอ่านโค้ด และจะทำให้แก้ไขโค้ดได้ยาก นอกจากคำสั่ง break และ continue แล้วยังมีคำสั่ง goto ซึ่งภาษาส่วนใหญ่เลิกใช้ ซึ่งทำให้การควบคุม Flow การทำงานซับซ้อนเกินไป และไม่เป็นลำดับ การอ่านโค้ดยากขึ้นและนำมาซึ่ง Error ในการเขียนโปรแกรมได้ง่าย
