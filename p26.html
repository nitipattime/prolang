สิ่งที่ภาษาการโปรแกรมต่างๆจะต้องพิจารณา

ในการเขียนโปรแกรมที่สามารถเขียนในลักษณะของการซ้อน เช่น ซ้อนฟังก์ชัน หรือซ้อนคำสั่งควบคุม ขอบเขตของตัวแปรนั้นจะจัดการอย่างไร ตัวอย่างเช่น 

int functionsomething( )
    {
            .......//บริเวณที่ 1
            int x = 10;
            .......//บริเวณที่ 2
    }

ขอบเขตของตัวแปรจะสัมพันธ์กับหน่วยนั้นอย่างไร

การมองเห็นตัวแปรจะเริ่มจากบรรทัดที่ประกาศตัวแปรในหน่วย หรือ สามารถมองเห็นได้จากหน่วยโดยไม่สนใจว่าตำแหน่งคำสั่งของการประกาศตัวแปรจะเป็นอย่างไร ในกรณีแรกนั้น การมองเห็นของตัวแปรจะเกิดขึ้นในบริเวณที่ 2 เท่านั้น ก็คือบริเวณหลังจากคำสั่งการประกาศตัวแปร ในกรณีที่สองนั้น การมองเห็นของตัวแปรจะเกิดขึ้นทั้งบริเวณที่ 1 และ 2 ดังนั้นตำแหน่งของการประกาศตัวแปรจึงไม่มีความสำคัญต่อการมองเห็นตัวแปร ขอบเขตของตัวแปรกับลักษณะของการเขียนคำสั่งซ้อน คำสั่งซ้อน เช่น ฟังก์ชันซ้อนในเงื่อนไข บล็อกซ้อนบล็อก ตัวอย่างฟังก์ชันซ้อนฟังก์ชันใน ภาษา PHP ไม่จำเป็นต้องประกาศตัวแปรก่อนใช้งาน จะถือว่า เมื่อเกิดคำสั่งกำหนดค่า จากโค้ดฟังก์ชัน y เป็นฟังก์ชันซ้อนภายในฟังก์ชัน x ดังนั้นบริเวณหน่วยของ y จึงอยู่ภายในหน่วยของ x ภาษาการโปรแกรมจะต้องพิจารณาการมองเห็นตัวแปรที่ไม่ได้ถูกประกาศในฟังก์ชันซ้อน 


    

    ภาษา PHP มีการจัดการขอบเขตในฟังก์ชัน โดยกำหนดให้สามารถมองเห็นได้เฉพาะตัวแปรที่ถูกประกาศในฟังก์ชันเท่านั้น  ดังนั้นจะไม่สามารถมองเห็นตัวแปร a ได้ในฟังก์ชันซ้อน
    แต่ PHP มีคำสั่งเพื่อกำหนดการมองเห็น คือ
