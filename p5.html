การนำโปรแกรมไปใช้

คือ วิธีการในการนำโปรแกรมไปใช้งาน เมื่อเขียนโปรแกรมเสร็จก็ต้องมีวิธีการนำไปใช้ สามารถแบ่งออกเป็น 3 วิธีการ

1. ในกลุ่มนี้ต้องคอมไพล์เพื่อขจัด error ก่อนทำการสร้าง โดยโค้ดที่ได้จากภาษาการโปรแกรมในกลุ่มนี้มักจะยึดติดกับสถาปัตยกรรมที่ใช้งาน ภาษาการโปรแกรมในกลุ่มนี้ได้แก่ , , , และ เป็นต้น เป็นคำที่ใช้เรียกโปรแกรมที่ทำหน้าที่คอมไพล์โปรแกรม (ที่ user เขียนขึ้น) และทำหน้าที่ในการสร้าง Machine code ซึ่งเป็นไฟล์นามสกุลแตกต่างกันไปในแต่ละภาษา ยกตัวอย่างเช่น ภาษา C มี C compiler สร้าง EXE file ซึ่งเป็น Machine code ที่สามารถรันได้กับสถาปัตยกรรมหนึ่งๆ (เครื่องที่ใช้เขียนโปรแกรม ณ ขณะนั้น)

2. ในกลุ่มนี้ไม่มี และไม่มีการสร้าง แต่มี ด้วย เพื่อแสดงผลลัพธ์จากการรันโปรแกรม จะไม่มีการสร้างรหัสเครื่องแต่ต้องมีโปรแกรมรันโค้ด (ที่เรียกว่า ) ซึ่งเป็นโค้ดที่ถือว่าอยู่ระหว่าง และ และถูกนำไปประมวลผลด้วยตัวตีความ (interpreter) ภาษาในกลุ่มนี้ได้แก่ , , และ ภาษาที่ใช้เขียนโปรแกรมบนเว็บส่วนใหญ่จะมีการนำไปใช้แบบ Interpretation

3. จะมีการสร้าง และอาจมี/ไม่มีการสร้าง ก็ได้ เช่น ปกติไม่มีการสร้างรหัสเครื่อง แต่มี (ไฟล์นามสกุล .class) ซึ่งเป็น โดยไบต์โค้ดนั้นถูกประมวลผลโดย ซึ่งทำหน้าที่เป็น หากต้องการสร้างรหัสเครื่องด้วยภาษา Java จะต้องเขียนโปรแกรมในลักษณะ จึงจะมีการสร้างรหัสเครื่อง การเขียนโปรแกรมในลักษณะ JIT นี้มักนำไปใช้ในการเขียนโปรแกรมกับฮาร์ดแวร์พิเศษที่สามารถประมวลผลโค้ดภาษา Java ได้ ภาพด้านล่างแสดงองค์ประกอบต่าง ๆ ในระบบ ซึ่ง เป็นการปรับปรุงโค้ดสื่อกลางเพื่อให้มีประสิทธิภาพมากขึ้น แล้วนำโค้ดนั้นไปสร้างเป็นรหัสเครื่องต่อไป การเขียนโปรแกรมสำหรับแพลตฟอร์มแอนดรอยด์ในโทรศัพท์มือถือในแอนดรอยด์บางเวอร์ชันมีการเรียกใช้ เพื่อสร้างรหัสเครื่อง (machine code) ที่สามารถรันได้บนโทรศัพท์มือถือในบางรุ่นโดยเฉพาะ

ข้อดีและข้อเสียของวิธีการนำโปรแกรมไปใช้

วิธีการนำไปใช้แบบ Compilation

ข้อดีข้อเสีย
  • โค้ดที่นำไปใช้ในเวลารันจะมีขนาดเล็ก
  • ีโค้ดที่นำไปใช้ในเวลารันสามารถประมวลผลได้เร็ว
  • โค้ดที่ผ่านการคอมไพล์จะนำไปรัน/ใช้งานบนเครื่องคอมพิวเตอร์หนึ่ง ๆ โดยเฉพาะ
  • เสียเวลาในการคอมไพล์นาน

วิธีการนำไปใช้แบบ Interpretation

ข้อดีข้อเสีย
  • การพัฒนาโปรแกรมทำได้เร็ว
  • โปรแกรมมีความยืดหยุ่น สามารถนำไปใช้งานได้หลายแพลตฟอร์ม (เช่น วินโดว์ ลินุกซ์)
  • การนำไปใช้มีลักษณะไดนามิค เช่น โค้ดของโปรแกรมจะถูกเลือกหรือนำไปใช้งานในสภาวะต่าง ๆ
  • เวลารันทำงานช้า เพราะเสียเวลาในการตรวจสอบและแก้ไขข้อผิดพลาด

วิธีการนำไปใช้แบบ Hybrid

ข้อดีข้อเสีย
  • โค้ดที่นำไปใช้ในเวลารันจะมีขนาดเล็ก
  • โค้ดที่นำไปใช้ในเวลารันสามารถประมวลผลได้เร็ว
  • โปรแกรมมีความยืดหยุ่น สามารถนำไปใช้งานได้หลายแพลตฟอร์ม
  • โค้ดที่ผ่านการคอมไพล์เพื่อสร้าง Machine code จะนำไปรัน/ใช้งานบนเครื่องคอมพิวเตอร์หนึ่ง ๆ โดยเฉพาะ จึงยากต่อการเคลื่อนย้ายโปรแกรม
  • เสียเวลาในการคอมไพล์นาน
