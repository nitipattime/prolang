คือความสัมพันธ์ที่เกิดขึ้นระหว่างสองสิ่งในเวลาหนึ่งๆ เช่น ตัวแปรกับชนิดข้อมูล ตัวแปรกับตำแหน่งในหน่วยความจำที่จัดเก็บค่าของตัวแปร ตัวแปรกับค่าข้อมูล เป็นต้น การผูกมัด จำแนกได้ 2 ลักษณะโดยพื้นฐานคือ

  1. คือ การผูกมัดที่เกิดขึ้นก่อนเวลารันและคงอยู่ไปตลอดจนกระทั่งการประมวลผลโปรแกรมเสร็จสิ้น เช่น การผูกมัดของชนิดข้อมูลกับตัวแปรในภาษาบางภาษาที่ไม่สามารถเปลี่ยนชนิดข้อมูลของตัวแปรได้ การผูกมัดของตัวแปรคงที่ (Constant variable) กับค่าข้อมูล การผูกมัดของตัวแปรกับตำแหน่งในหน่วยความจำที่ไม่สามารถเปลี่ยนแปลงตำแหน่งได้
  2. คือ การผูกมัดที่เกิดขึ้นในเวลารันแต่สามารถเปลี่ยนแปลงการผูกมัดได้ในเวลารัน เช่น การผูกมัดของตัวแปรกับค่าข้อมูลที่สามารถเปลี่ยนแปลงค่าข้อมูลได้ การผูกมัดของตัวแปรกับหน่วยความจำที่สามารถเปลี่ยนแปลงตำแหน่งอ้างอิงในหน่วยความจำได้

การผูกมัด (Binding) มีความสัมพันธ์กับเวลาที่เกิดการผูกมัด (Binding time) การผูกมัดเกิดขึ้นได้ตลอดเวลาในระหว่างการรันโปรแกรม และสามารถพิจารณาเวลาผูกมัดได้หลายช่วง เช่น

  • Language design time (เวลาออกแบบภาษา)
  • Language implementation time (เวลานำไปใช้)
  • Compile time (เวลาคอมไพล์)
  • Link time (เวลาเชื่อมโยง)
  • Load time (เวลาโหลด) และ
  • Run time (เวลารัน)

เวลาข้างต้นเรียงลำดับตาม เวลาที่เกิดขึ้นก่อน และหลัง เช่น เวลาออกแบบภาษาเกิดขึ้นก่อนเวลานำไปใช้

เช่น การออกแบบชิ้นส่วนต่างๆ (construct) ของคำสั่งในภาษาโปรแกรม ซึ่งเกิดการผูกมัดระหว่างสัญลักษณ์ตัวดำเนินการและการดำเนินการ

- การลบ

คำสั่งเงื่อนไข if-else ผูกมัดกับการดำเนินการเพื่อควบคุมการทำงานของโปรแกรม
คำสั่ง while ผูกมัดกับการดำเนินการเพื่อควบคุมการทำงานของโปรแกรม

เกี่ยวข้องกับการผูกมัดระหว่าง ชนิดข้อมูลและรูปแบบการจัดแสดงข้อมูลในหน่วยความจำ เช่น Floating-point number ผูกมัดกับ

เกี่ยวข้องกับการเขียนโปรแกรมที่มีการอ้างอิงไปยังโปรแกรมอื่น ดังนั้นก่อนที่จะคอมไพล์จะเกิดการผูกมัดเพื่อเชื่อมโยงโค้ดที่เกี่ยวข้องเข้าด้วยกัน เช่น ตัวอย่างในภาพที่ 3.3 Assembler (โปรแกรมทำหน้าที่ในการรวมซอร์สโค้ด) จะทำการคอมไพล์ไฟล์ซอร์สโค้ดต่างๆ ได้ Object file แล้วLinker (โปรแกรม) จะทำการรวมโค้ด Object file (เป็นไฟล์แมชชีนโค้ดแต่ยังไม่สามารถรันได้) นั้นเข้าด้วยกันรวมถึง Library ต่าง ๆ ที่โปรแกรมมีการเรียกใช้ แล้วจึงทำการสร้าง Executable file ซึ่งเป็นแมชชีนโค้ดซึ่งสามารถนำไปรันบนระบบต่อไป ในภาษา C นั้น Object file คือ Intermediate code ของแต่ละไฟล์ซึ่งจะถูกนำไปรวมเพื่อสร้าง .EXE ไฟล์ต่อไป ซึ่ง Assembler และ Linker ต่างอยู่ในคอมไพเลอร์โปรแกรม การผูกมัดในเวลาเชื่อมโยงในลักษณะอื่น เช่น การเรียกใช้ #include ในภาษา C หรือ การใช้คำสั่ง include, require ในภาษา PHP

คือ เวลาผูกมัดที่เกิดขึ้นเมื่อมีการเข้าถึงข้อมูลในเมมโมรีก่อนประมวลผลคำสั่งในหน่วยประมวลผล (CPU) ตัวอย่างเช่น การโหลดตัวแปรถาวร/ตัวแปรสตาติก (Static variable) จะเกิดการผูกมัดไปยังตำแหน่งทางกายภาพ (Physical address) ตำแหน่งหนึ่งตลอดการประมวลผลโปรแกรม ดังนั้นการเข้าถึงตัวแปรถาวรจะได้ค่าล่าสุดที่ถูกจัดเก็บ

คือ การผูกมัดที่เกิดในเวลารันหรือระหว่างประมวลผลโปรแกรม เช่น ตัวแปรกับค่าข้อมูล (จะรู้ว่าตัวแปรมีค่าในเวลารัน) ตัวแปรกับตำแหน่งในเมมโมรี (จะรู้ตำแหน่งของตัวแปรในหน่วยความจำในเวลารัน) คำสั่งหนึ่งคำสั่งจะเกี่ยวข้องกับการผูกมัดได้หลายลักษณะ เช่น int x = 10; - ตัวแปร x มีการผูกมัดไปยังชนิดข้อมูลอินเทเจอร์ในเวลาคอมไพล์ - ตัวแปร x มีการผูกมัดไปยังค่าข้อมูลคือ 10 ในเวลารัน - ตัวแปร x มีการผูกมัดไปยังตำแหน่งในเมมโมรีในเวลารัน
