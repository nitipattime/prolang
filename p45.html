การแปลงชนิดข้อมูล

มีการจัดการ 2 แบบ


  1. เป็นการดำเนินการด้วยหน่วยประมวลผลข้อมูล (CPU) ในระหว่างรัน

  2. เป็นการดำเนินการด้วยนักพัฒนาโปรแกรม เช่น ส่วนใหญ่จะทำเมื่อต้องการแปลงขนาดของชนิดข้อมูลที่มีขนาดใหญ่ไปแสดงด้วยชนิดข้อมูลที่มีขนาดเล็กกว่า

ตัวอย่าง Type Casting ใน C 

#include 
        
        main() {
        
           int sum = 17, count = 5;
           double mean;
        
           mean = (double) sum / count;
           printf("Value of mean : %f\n", mean );
        }

การแปลงชนิดข้อมูลแบบอ้อม (Implicit Type Conversion)
จะเกิดขึ้นในโปรแกรมที่มีการดำเนินการกับนิพจน์ที่มีการใช้ตัวแปรที่มีชนิดข้อมูลที่แตกต่างกัน แต่สามารถนำมาเป็นตัวถูกดำเนินการสำหรับตัวดำเนินการหนึ่ง ๆ ได้ ภาษาการโปรแกรมจัดการการแปลงชนิดข้อมูลของนิพจน์ในเวลารัน เช่น โค้ดในภาษา Java 

 double d;
         long l;
         int i;
         if (d > i) d = i;
         if (i > l) l = i;
         if (d == l) d *= 2;

นิพจน์ที่เกี่ยวข้องกับเงื่อนไข if เป็นการเปรียบเทียบตัวแปรซึ่งมีชนิดข้อมูลที่แตกต่างกัน ดังนั้นภาษาการโปรแกรมจำดำเนินการทำการแปลงชนิดข้อมูลเพื่อเปรียบเทียบ โดยพิจารณาเงื่อนไขที่ภาษาการโปรแกรมกำหนด 2 รูปแบบ ได้แก่

  1. การแปลงชนิดข้อมูลไปเป็นอีกชนิดข้อมูลหนึ่ง โดยที่ปลายทางอาจไม่สามารถเก็บค่าข้อมูลของต้นทางได้ จะทำให้ค่าข้อมูลขาดหายหรือไม่ครบถ้วน ชนิดข้อมูลที่มีขนาดใหญ่ ---> ชนิดข้อมูลที่มีขนาดเล็ก
  2. การแปลงชนิดข้อมูลไปเป็นอีกชนิดข้อมูลหนึ่งโดยที่ปลายทางสามารถเก็บค่าข้อมูลของต้นทางได้ทั้งหมด จะทำให้ค่าข้อมูลนั้นครบถ้วนแต่อาจทำให้เปลืองพื้นที่ในการใช้งานในหน่วยความจำ ชนิดข้อมูลที่มีขนาดเล็ก ---> ชนิดข้อมูลที่มีขนาดใหญ่

หากภาษา Java ทำการแปลงชนิดข้อมูลแบบอ้อม แบบ Narrowing conversion โดยภาษากำหนดกฎการแปลงแบบ Narrowing conversion (ดูตาราง) จากล่างขึ้นบน เช่น double -> float -> long -> int -> short -> char -> byte -> boolean
(d > i) จะแปลง Double (d) ไปเป็น Integer (i) เพื่อประเมินผลการเปรียบเทียบ
(i > l) จะแปลง Long (l) ไปเป็น Integer (i) เพื่อประเมินผลการเปรียบเทียบ
(d == l) จะแปลง Double(d) ไปเป็น Long (l) เพื่อประเมินผลการเปรียบเทียบ

หากภาษา Java ทำการแปลงชนิดข้อมูลแบบอ้อม แบบ Widening conversion โดยภาษากำหนดกฎการแปลงแบบ Widening conversion (ดูตาราง) จากล่างขึ้นบน เช่น boolean -> byte -> char -> short -> int -> long -> float -> double
(d > i) จะแปลง Integer (i) ไปเป็น Double (d) เพื่อประเมินผลการเปรียบเทียบ
(i > l) จะแปลง Integer (i) ไปเป็น Long (l) เพื่อประเมินผลการเปรียบเทียบ
(d == l) จะแปลง Long (l) ไปเป็น Double(d) เพื่อประเมินผลการเปรียบเทียบ

ข้อผิดพลาดจากการแปลงด้วย Narrowing conversion จะเกิดขึ้นเมื่อค่าข้อมูลหายไปจากการแปลง

ข้อผิดพลาดที่เกิดจากการแปลงชนิดข้อมูลแบบ Implicit conversion จะแจ้งข้อผิดพลาด Incompatible type ด้วยข้อความ “Type mismatch” เช่น โค้ดในภาษา Java 

     float num = 3.14;

เนื่องจาก 3.14 ถูกนำไปใช้ด้วยชนิดข้อมูลแบบ double ในขนะที่ num คือ float ภาษา Java มีการจัดการการแปลงชนิดข้อมูลแบบ Widening Conversion ดังนั้นเมื่อต้องการแปลงต้องทำ Type Casting การแปลงชนิดข้อมูลแบบประกาศ (Explicit Type Conversion) คือการแปลงชนิดข้อมูลด้วยการกำหนดคำสั่งจากนักเขียนโปรแกรม เช่น การทำ Type Casting Type Casting ในภาษา Java 

     float num = (float) 3.14;
หรือ 
    float num = 3.14f;
        
float height = 1.74f; // Explicit conversion
        double MaxH = height; // Implicit
        double MinH = (double) height; // Explicit
        float AHeight = (float) MaxH; // Explicit
