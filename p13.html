ความสามารถในการอ่านของภาษาการโปรแกรม

จะพิจารณาโดยผู้อ่านโปรแกรมเป็นหลัก หรือนักพัฒนาโปรแกรม คุณลักษณะที่ดีของโปรแกรมในด้านการอ่านจะต้องอ่านโค้ดง่าย นักพัฒนาโปรแกรมสามารถเข้าใจแนวคิดในการเขียนโปรแกรม ความจำเป็นที่จะต้องประเมินความสามารถในด้านนี้ ก็เพราะเมื่อมีการนำโปรแกรมไปใช้งาน อาจจะต้องทำการแก้ไขโค้ดโดยนักพัฒนาโปรแกรมคนอื่น ถ้าภาษาที่เลือกใช้ในการเขียนโปรแกรมอ่านง่าย ก็จะแก้ไขโค้ดได้ง่าย เสร็จเร็ว และมีข้อผิดพลาดน้อย ภาษาที่อ่านโปรแกรมได้ง่าย มักมีคุณลักษณะดังต่อไปนี้

  • คำสั่งและตัวดำเนินการที่มีในภาษาการโปรแกรมง่ายและไม่ซับซ้อน
  • ที่เหมาะสมที่ช่วยให้อ่านโค้ดได้ง่าย
  • ที่เหมาะสม

คำสั่งและตัวดำเนินการที่มีในภาษาการโปรแกรมง่ายและไม่ซับซ้อน

ภาษาที่มีคำสั่งไม่มากนัก มักจะอ่านง่าย แต่อาจด้อยในเรื่องการเขียนโปรแกรมเพื่อแก้ปัญหาที่ซับซ้อน สมมติว่ามีภาษาสมมติ (Anynomous) มีโอเปอเรเตอร์บวก + แทนการดำเนินการบวก และภาษา C มีโอเปอเรเตอร์ ++ แทนการดำเนินการบวกแบบเพิ่มค่า (Increment operator) และโอเปอเรเตอร์ + แทนการบวก 

ภาษา A
        x = 10;
        x = x + 1;

ภาษา C สามารถเขียนคำสั่งเพื่อเพิ่มค่าของ x ได้ถึง 3 รูปแบบคือ

รูปแบบที่ 1
        x = 10;
        x++;
        
รูปแบบที่ 2
        x = 10;
        ++x;
        
รูปแบบที่ 3
        x = 10;
        x = x + 1;

ทั้งสองภาษาการโปรแกรม ทำให้ x เพิ่มค่าเป็น 11 และจะเห็นได้ว่าภาษา C เขียนได้ถึง 3 รูปแบบ เมื่อพิจารณาการอ่านโค้ด เราจะเห็นว่าภาษา A อ่านได้ง่ายกว่า เพราะไม่ต้องทำความเข้าใจการกำหนดตัวดำเนินการในหลายๆลักษณะอย่างที่ภาษา C มี เมื่อพิจารณาจากการเขียน เราจะเห็นว่าภาษา C เขียนได้ง่ายกว่า เพราะมีรูปแบบให้เลือกเขียนถึง 3 รูปแบบ และเขียนคำสั่งได้สั้นกว่าเมื่อเลือกใช้ Increment operator จากตัวอย่างข้างต้น มี และ ซึ่งมี คือสามารถเขียนได้หลายแบบเช่น กำหนดไว้หน้าตัวถูกดำเนินการ หรือกำหนดไว้หลังตัวดำเนินการ 

ภาษา C
        x = 10;
        y = x++;
        
ภาษา C
        x = 10;
        y = ++x;

กรณีนี้จะพิจารณาได้ว่า การอ่านโค้ดเริ่มจะยากขึ้น เพราะนักเขียนโปรแกรมต้องเข้าใจถึงลำดับการประเมินผลของตัวดำเนินการที่เข้ามาเกี่ยวข้องด้วย ลองหาคำตอบดูสิ ว่า y มีค่าอย่างไร จากการเขียนคำสั่งข้างต้น

มักจะทำให้การอ่านโค้ดอ่านยาก แต่การเขียนโปรแกรมนั้นง่าย

ชนิดข้อมูล

ภาษาที่อ่านง่ายควรมีชนิดข้อมูลที่เหมาะสม ช่วยให้อ่านโค้ดเข้าใจได้ง่าย สมมติภาษา A ไม่มีชนิดข้อมูลบูลีน (boolean) มีเฉพาะอินเทเจอร์ (integer) ถ้าอยากจะเขียนโปรแกรมเพื่อกำหนดสถานะการหมดเวลา (timeout) จะเขียนได้ดังต่อไปนี้ 

ภาษา A
        timeout = 0;

และในภาษา C มีชนิดข้อมูลบูลีน ซึ่งจะเขียนได้ดังนี้

timeout = true;

ดังนั้นภาษา A จะอ่านยากกว่า เพราะนักพัฒนาต้องทำความเข้าใจเพิ่มว่า 0 สื่อถึงสถานะการหมดเวลาเกิดขึ้นจริง ส่วนภาษา C นั้นนักเขียนโปรแกรมสามารถเข้าใจได้ทันที สรุป ภาษาที่มีความสามารถในการอ่านควรมีชนิดข้อมูลที่เพียงพอต่อการใช้งาน

อีกตัวอย่างหนึ่งคือ ยกตัวอย่างเช่นโค้ดในภาษา C ชนิดข้อมูลแบบ Enumeration ทำให้การอ่านโค้ดอ่านง่าย จะเห็นได้จากตัวอย่างว่าสัปดาห์ประกอบด้วยวันต่างๆ ในการทำงานของภาษาการโปรแกรมมักมีการกำหนดข้อความของชนิดข้อมูลแบบ Enumeration ไปเป็น Primitive data type เช่นในภาษา C นั้นค่าที่กำหนดจะถูกนำไปกำหนดด้วย Integer 

// An example program to demonstrate working
        // of enum in C
        #include
          enum week{Mon, Tue, Wed, Thur, Fri, Sat, Sun};
        int main()
        {
            enum week day;
            day = Wed;
            printf("%d",day);
            return 0;
        }

ลองใช้ C Online compiler รันโค้ดนี้เพื่อดูคำตอบ สิ่งที่น่าสนใจค้นหาเพิ่มเติมคือ เมื่อกำหนด Enumeration แล้ว สามารถนำค่าของ Enumeration นั้นไปทำการคำนวณได้หรือไม่ ? หรือภาษาการโปรแกรมต้องการออกแบบเพื่อให้เป็นประโยชน์สำหรับการอ่านโค้ดเท่านั้น

ความยาวของการตั้งชื่อตัวแปร
สมมติว่าภาษา A อนุญาตให้กำหนดชื่อตัวแปรโดยมีความยาวเพียง 3 ตัวอักขระและใช้ตัวอักษร a-z เท่านั้น ในขณะที่ภาษา Java สามารถกำหนดชื่อตัวแปรได้ยาวไม่จำกัด หากต้องการตั้งชื่อตัวแปรแทนรหัสนักศึกษา ภาษา A

int stu;
ภาษา Java 
int studentid;

การสื่อความหมายของภาษา Java จะทำได้ดีกว่า เพราะไม่มีข้อกำหนดในเรื่องของความยาวของชื่อตัวแปร ดังนั้นสามารถกำหนดชื่อที่สื่อความหมายเพื่อสร้างความเข้าใจได้ การอนุญาตให้ใช้คำพิเศษมาตั้งชื่อตัวแปร

สมมติว่าภาษาการโปรแกรมอนุญาตให้ใช้ มาตั้งชื่อตัวแปรได้ ในขณะที่อีกภาษาหนึ่งไม่อนุญาตให้ใช้ เช่น

ภาษา A
        int float;

อาจทำให้การอ่านโค้ดเริ่มสับสนได้ ซึ่ง float คือชื่อตัวแปรแต่มีชนิดข้อมูลเป็นอินเทเจอร์ (เลขจำนวนเต็ม)
ภาษาที่มีความสามารถในการอ่านไม่ควรอนุญาตให้ใช้ Special word ในการกำหนด Identifier หรือหากอนุญาตก็ไม่ควรให้ใช้มากจนเกินไป ตัวอย่างโค้ดภาษา 

MODULE Circle_Operations
        IMPLICIT NONE
        CONTAINS
        FUNCTION Area(radius)
        REAL :: Area
        REAL, INTENT(IN) :: Radius
        Area = Radius**2 * 3.14159
        END FUNCTION Area
        FUNCTION Circumference(radius)
        REAL :: Circumference
        REAL, INTENT(IN) :: Radius
        Circumference = Radius * 2 * 3.14159
        END FUNCTION Circumference
        END MODULE Circle_Operations
        PROGRAM Circle
        USE Circle_Operations
        IMPLICIT NONE
        REAL :: r=5.0
        PRINT *,"Area=",Area(r)
        PRINT *,"Circumference=",Circumference(r)
        END PROGRAM Circle
