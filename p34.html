การประเมินผลนิพจน์แบบ Short circuit

ภาษาการโปรแกรม อาจมีการประเมินผลนิพจน์แบบ Short circuit กล่าวคือ การประเมินผลนิพจน์ หนึ่งๆ อาจไม่จำเป็นที่จะต้องทำการประเมินผลนิพจน์ย่อยทั้งหมด ก็สามารถส่งค่าผลลัพธ์ได้แล้ว อาจพิจารณาการคำนวณบางประเภทในทางคณิตศาสตร์ เช่น 

x    =  b x c  x  d  / e

ในทางคณิตศาสตร์ หากนำค่า 0 ไปคูณกับค่าของตัวแปรใดก็ตาม ผลลัพธ์จะได้ 0 เสมอ ดังนั้นถ้าพบว่า b หรือ c หรือ d มีค่าเท่ากับ 0 ก็ไม่จำเป็นต้องประเมินผลนิพจน์ในส่วนที่เหลือ สามารถส่งค่าผลลัพธ์คือ 0 ได้ทันที

ในทางคอมพิวเตอร์ หากภาษาการโปรแกรมมีการสนับสนุนการดำเนินการแบบลัดวงจรนี้ จะเป็นประโยชน์ต่อเวลาที่ใช้ในการรันโปรแกรม ทำให้รันได้เร็วขึ้น โดย CPU ไม่จำเป็นต้องเสียเวลาในการดำเนินการกับนิพจน์บางส่วนที่ไม่จำเป็น

สิ่งที่ภาษาการโปรแกรมจะต้องพิจารณาคือ โอเปอเรเตอร์ใดบ้างที่ภาษาการโปรแกรมจะสนับสนุนการทำงานลัดวงจร อาจพิจารณาในเบื้องต้นสำหรับตัวดำเนินการต่อไปนี้

  • ตัวดำเนินการด้วยโอเปอเรเตอร์เปรียบเทียบ
  • ตัวดำเนินการด้วยโอเปอเรเตอร์เปรียบเทียบ

ตัวอย่างที่ 1

if (A AND B AND C)  then.....
        else......
เช่นในภาษา C 
if ((a == 0) AND (b <10) AND (c > d))
           result = 50;
        else
           result = 100;

อธิบายโค้ด


หากในเวลารันพบว่านิพจน์ (a==0) เป็นเท็จ ดังนั้นภาษาการโปรแกรมอาจทำการประเมินผลนิพจน์แบบลัดวงจรและส่ง false เป็นผลลัพธ์ของการพิจารณาเงื่อนไขนี้เลยโดยไม่ต้องประเมินผลนิพจน์อื่นๆที่เหลือ เพราะนิพจน์ใดนิพจน์หนึ่งเป็นเท็จก็จะทำให้ผลลัพธ์เป็นเท็จทันที และเป็นไปในแนวทางเดียวกันกับ OR หากนิพจน์ใดนิพจน์หนึ่งเป็นจริง ก็จะทำให้ได้ผลลัพธ์เป็นจริง เช่น ตัวอย่างที่ 2

if (A OR B OR C)  then.....else......
เช่น 
if ((a == 0) OR (b <10) OR (c > d))
            result = 50;
        else
            result = 100;

อธิบายโค้ด


หากในเวลารัน ประเมินผลนิพจน์ได้ว่า นิพจน์ใดนิพจน์หนึ่งเป็น จริง ก็จะทำให้เงื่อนไขนี้เป็นจริง เช่น เมื่อทำการประเมินผลนิพจน์ (a==0) เป็นจริง ก็สามารถรีเทิร์นผลลัพธ์ได้เลย
