คุณลักษณะนิพจน์ที่มีความโปร่งใส

นิพจน์ที่มีความโปร่งใส หรือ นิพจน์ที่มีคุณลักษณะ Referential Transparency คือนิพจน์ (E) ที่สามารถแทนที่ค่า ตัวแปร หรือฟังก์ชันในนิพจน์ แล้วไม่ทำให้ค่าของนิพจน์(Ev) เปลี่ยนแปลงไป 
Ev  =  E
        โดย
        E = {Literal value, variable, function}
เช่น 
a + b = a + 10

หากแทนค่า b ด้วย 10 ก็จะทำให้ประโยคคำสั่งด้านซ้ายและขวามีค่าเท่ากับ ยกตัวอย่างเช่นเมื่อเขียนโปรแกรมคำสั่ง 

b = 10;
        x = a + b;
จะมีค่าเท่ากับ 
x = a + 10;

ข้อผิดพลาดในการเขียนโปรแกรม
ในบางกรณีนักพัฒนาคิดว่าค่าที่ได้จากนิพจน์มีค่าเท่ากัน เนื่องจากนิพจน์ที่ตนเขียนนั้นมีรูปแบบของคำสั่งที่มองผิวเผินแล้วมีคุณลักษณะ Referential Transparency ยกตัวอย่างโค้ดภาษา C เช่น

a = 20;   //สมมติให้ a เป็นตัวแปรที่มองเห็นได้ในฟังก์ชัน
        x  = a + add(10, 30);    //นิพจน์ E1
        y = a + add(10,30);     //นิพจน์ E2
                                //นักพัฒนาโปรแกรมจะดากหมายว่าตัวแปร x และ y ต้องมีผลลัพธ์เท่ากัน  แต่ในการประเมินผลที่เกิดขึ้นผลลัพธ์ไม่เท่ากัน
        int add(int   x, int y){
               int sum;
               a  = 10;
               sum = x + y;
               return sum;
        }

อธิบายโค้ด


พิจารณาการเขียนโปรแกรมเบื้องต้น สมมติให้ลำดับการดำเนินการของพิจน์ที่เครื่องคอมพิวเตอร์คำนวณเป็นดังนี้ อ่านค่าของตัวโอเปอแรนด์ a ดำเนินการเป็นลำดับตามเครื่องหมายโอเปอเรเตอร์ ซึ่งการเรียกฟังก์ชันจะเกิดขึ้นหลังจากอ่านค่าของโอเปอแรนด์ a แล้ว ทำการหาผลบวก นักเขียนโปรแกรมจะเข้าใจว่า
นิพจน์ E1 และ E2 เมื่อมองรูปแบบโค้ดคำสั่งก็น่าจะได้ผลลัพธ์แบบเดียวกัน คือ ค่าของ x จะเท่ากับค่าของ y นักเขียนโปรแกรมอาจเข้าใจว่าผลลัพธ์ x และ y เท่ากัน เพราะตนมองเห็น a เป็น 20 และคาดหมายผลลัพธ์คือ 60 แต่ความเป็นจริงนิพจน์ E1 และ E2 ได้ผลลัพธ์เท่ากันคือ 50 เพราะมีการเรียกฟังก์ชัน add ก่อนซึ่งฟังก์ชันทำการเปลี่ยนแปลงค่าของตัวแปร a

ดังนั้นจึงสรุปได้ว่านิพจน์ E1 และ E2 มีคุณลักษณะ Referential Transparency ระหว่างกัน แต่จะเห็นว่าเมื่อการเรียกฟังก์ชันมีปัญหา Side effect เกิดขึ้น ค่าที่นักพัฒนาโปรแกรมคาดหวังอาจไม่เท่ากับค่าที่ เกิดจากการประมวลผลจริง

ตัวอย่างกรณีที่ทำให้นิพจน์ E1 และ E2 ไม่มีคุณลักษณะ Referential Transparency
