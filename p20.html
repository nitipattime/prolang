การผูกมัดตัวแปรกับตัวดำเนินการ

ถูกนำไปพิจารณาในการทำ เพื่อพิจารณาการใช้งานร่วมกันระหว่างโอเปอแรนด์และโอเปอเรเตอร์ ในการประเมินผลนิพจน์ การผูกมัดระหว่างตัวแปรกับตัวดำเนินการเกี่ยวข้องกับการตรวจสอบชนิดข้อมูล (Type checking) ในการคอมไพล์จะมีการตรวจสอบชนิดข้อมูล ซึ่งจะทำการตรวจสอบว่าโค้ดที่เขียนนั้นตัวถูกดำเนินการมีชนิดข้อมูลที่สอดคล้องกับตัวดำเนินการหรือไม่ เช่น ในภาษา C ตัวดำเนินการลบ (-) สามารถใช้งานได้กับตัวถูกดำเนินการที่มีชนิดข้อมูลแบบ Integer และ float ดังนั้นคำสั่ง 

A = B – C;

คอมไพเลอร์จะทำการตรวจสอบว่า B และ C มีชนิดข้อมูลแบบ Integer หรือ float หรือไม่ หากใช่จะผ่านการตรวจสอบและหากไม่ใช่จะแสดงข้อผิดพลาดจากการตรวจสอบชนิดข้อมูล การตรวจสอบชนิดข้อมูลจะมีความซับซ้อนมากขึ้นเมื่อตัวดำเนินการสามารถนำไปใช้งานได้มากกว่าหนึ่งวัตถุประสงค์ ตัวอย่างโค้ด PHP

  <?php
           $a = 10;
           $b = "Hello";
           $c = "Everybody";
           $d = $a +  50 ;
           $e = $b +  $c;      //Type error
          ?>

ข้อผิดพลาดที่เกิดจาก Type checking เรียกว่า โค้ด PHP ต้องแก้ไขตัวดำเนินการ จาก $e = $b + $c; เป็น $e = $b . $c;

อธิบายโค้ด

Type error ในภาษา C/C++

    อาจจำแนกเบื้องต้นได้แก่
  • ข้อผิดพลาดจากการเขียนรูปแบบประโยคที่ไม่ถูกต้องตามหลักการของภาษา การประกาศตัวแปร
  • ข้อผิดพลาดที่เกิดขึ้นในเวลารัน เช่น การหารด้วย 0
  • ข้อผิดพลาดจากการเขียนคำสั่งแบบผิดๆ โดยอาจเป็นความหลงลืมของนักเขียนโปรแกรม
  • ข้อผิดพลาดจากการอ้างอิงไปยังฟังก์ชัน หรือไฟล์ที่ไม่มี
  • ข้อผิดพลาดจากการตีความคำสั่ง
