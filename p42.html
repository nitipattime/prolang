ชนิดข้อมูลประเภทต่างๆ

ชนิดข้อมูลพื้นฐาน (Primitive datatype)

ทุกภาษาการโปรแกรมต้องกำหนดชนิดข้อมูลพื้นฐาน ชนิดข้อมูลพื้นฐานอาจนำไปพัฒนาเพื่อสร้างชนิดข้อมูลอื่นๆ ได้ (non-primitive datatype) การออกแบบมักจะสัมพันธ์กับสถาปัตยกรรม เช่น Integer อาจมีทั้ง 64bits หรือ 32bits การทำงานกับชนิดข้อมูลประเภทนี้ไม่ต้องจัดการในระดับซอฟต์แวร์มากนัก (เช่น การเข้าถึงข้อมูลอาจไม่ต้องเขียนฟังก์ชันช่วยในระดับสถาปัตยกรรม(ในระดับ Hardware))

: Primitive datatype

พิจารณาการเข้าถึงข้อมูลระหว่างชนิดข้อมูลแบบ Integer ซึ่งเป็น Primitive datatype กับ Array ซึ่งเป็น Complex datatype จะเห็นว่าการเข้าถึงข้อมูลของ Complex datatype จะต้องมีการเขียนโปรแกรมช่วยในระดับสถาปัตยกรรมเพื่อเข้าถึงข้อมูลในหน่วยความจำ มากกว่าการทำงานกับ Integer

: Primitive datatype

แสดง Real number ภาษาส่วนใหญ๋ floating point ครอบคลุม float และ double มีการแสดงค่าข้อมูลด้วยมาตรฐานบางอย่างเช่น IEEE Floating-Point Standard 754

: Primitive Data Types

บางภาษาสนับสนุน complex type เช่น Imaginary number เช่น ภาษาไพธอน รูปแบบคำสั่งการกำหนด Imaginary number ประกอบด้วย Real part และ Imaginary part (7 + 3j) 7 เป็น real part และ 3 เป็น imaginary part อ่านเพิ่มเติม http://mathworld.wolfram.com/ImaginaryPart.html

: Primitive Data Types

เป็น ชนิดข้อมูลที่ออกแบบเพื่อแอพพลิเคชันทางธุรกิจ ที่จัดการเรื่องเงิน โดยเฉพาะ กำหนดกลุ่มของบิตแทน digit ของเลขฐาน 10 ภาษาที่มีชนิดข้อมูลประเภทนี้ได้แก่ COBOL, C# ตัวอย่าง C# http://csharpindepth.com/Articles/General/Decimal.aspx ข้อเสีย คือ ใช้หน่วยความจำเปลือง และค่อนข้างจำกัดค่าข้อมูลที่สามารถแสดงได้ เช่น ถ้ามี 16 บิต แต่ละดิจิตใช้ 4 บิต จะแทนตัวเลขได้เกียง 4 หลัก ทำให้ช่วงค่าของข้อมูลน้อยตามไปด้วย Big-endian format สามารถอ่านได้เพิ่มเติมที่https://www.geeksforgeeks.org/little-and-big-endian-mystery/

: Primitive Data Types

ประโยชน์ในการใช้งานคือ การเพิ่มความสามารถในการอ่านโค้ด สมมติ ต้องการเขียนคำสั่ง เพื่อกำหนดสสถานะว่า หมดเวลา กับยังไม่หมดเวลา เราอาจเขียนด้วย 

timeout = true
        timeout = false

จะสื่อความหมายดีกว่า

timeout = 1
        timeout = 0

เนื่องจากค่าที่ต้องการแสดงมีค่าเป็น 0 หรือ 1 เท่านั้น จึงต้องการแค่บิตเดียว แต่ในความเป็นจริงในภาษาการโปรแกรมมักจะแทน Boolean ด้วย Byte

ค่าข้อมูลคือกลุ่มของตัวอักขระ (character) การออกแบบในภาษาการโปรแกรมต่างๆ อาจจะต้องพิจารณาว่า จะสร้างให้เป็น Primitive datatype หรือเป็นอาเรย์ ใน C, C++:

char mystring[50];    //array of characters.
        char *s0 = "hello world";
        char s1[] = "hello world";

    สิ่งที่ควรพิจารณาคือ
  • ความยาวของสตริงจะเป็นแบบ static (คงที่) หรือ ไดนามิค (ไม่จำเป็นต้องระบุขนาดความยาวของสตริง
  • มีการดำเนินการอะไรบ้างที่สามารถทำงานกับสตริงได้ เช่น การกำหนดค่า (assignment) การสำเนาค่า (copying) การเปรียบเทียบ (comparison) การต่อสตริง (Catenation) การจับคู่รูปแบบสตริง (Pattern matching)
    การมีการดำเนินการ (operation) จำนวนมากในภาษาการโปรแกรมจะทำให้ง่ายต่อการเขียนโปรแกรม ทำให้เพิ่มความสามารถในด้าน Writability
  • ประเภทความยาวของสตริง
    • มีข้อมูลกำหนดความยาวที่ถูกจัดเก็บใน Runtime descriptor เมื่อใช้ฟังก์ชันหาความยาวของสตริงจะได้ค่าตามที่กำหนดเสมอ
    • ความยาวของสตริงเปลี่ยนแปลงตลอดเวลา อาจเกิด overhead ในการจัดสรรพื้นที่และคืนพื้นที่ได้ (เกิดการรบกวนหน่วยความจำบ่อย) เช่น ในภาษา JavaScript, Perl, Java

      ตัวอย่างการเขียนโปรแกรมเพื่อทำงานกับสตริงในภาษา Java 

      public class HelloWorld{
        
             public static void main(String []args){
        
                String mystr = "Hi";
                System.out.println(mystr.length());
                mystr = "Hello world";
                System.out.println(mystr.length());
             }
        }

    • มีการกำหนดขนาดสูงสุด แต่ความยาวของสตริงสามารถเปลี่ยนแปลงได้ตลอดเวลาแต่ต้องไม่เกิดขนาดที่กำหนดสูงสุด สตริงในภาษา C และ C++ จัดอยู่ในประเภทนี้
