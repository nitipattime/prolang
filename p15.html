ประเมินด้านความเชื่อถือได้

การประเมินความสามารถในด้าน

ภาษาการโปรแกรมที่ดีนั้นอาจพิจารณาได้ 3 ประเด็นหลักๆ ได้แก่

  • เมื่อนำไปเขียนโปรแกรมแล้วไม่ควรมีคำสั่งที่ทำให้เกิด ได้ง่ายเป็นจำนวนมาก

    ภาษาการโปรแกรมมีขั้นตอนของเพื่อตรวจหาข้อผิดพลาด แต่คำสั่งบางอย่างในการทำงานนั้นไม่สามารถหาข้อผิดพลาดได้ในเวลาคอมไพล์ เช่น คำสั่งที่เกี่ยวข้องกับการใช้งานตัวแปรพอยน์เตอร์

  • มี เพื่อตรวจจับข้อผิดพลาดในลักษณะต่างๆ

    ภาษาการโปรแกรมที่มีคำสั่งในการตรวจจับข้อผิดพลาดในระหว่างเวลารัน จะช่วยลดข้อผิดพลาดของโปรแกรมได้ดีกว่าภาษาการโปรแกรมที่ไม่มีคำสั่งประเภทนี้ เช่น ในภาษา Java มีคำสั่งเกี่ยวข้องกับการตรวจจับข้อผิดพลาดที่เกิดกับการใช้งานรีซอร์ส (ไฟล์ ฐานข้อมูล) จะทำให้นักพัฒนาโปรแกรมสามารถวางแผนแก้ไขข้อผิดพลาดนั้นได้ในเวลารันและจะทำให้สามารถรันโปรแกรมต่อไปได้

  • มีคำสั่งที่ไม่มากหรือไม่ซับซ้อนในการใช้งาน

    ถ้าภาษามีคำสั่งจำนวนมาก หรือคำสั่งนั้นมีการใช้งานที่ซับซ้อน ก็อาจนำมาซึ่งข้อผิดพลาดได้ง่าย

ที่

ได้แก่

  1. การเขียนคำสั่งที่ผิดโดยใช้ไวยากรณ์ (grammar) ของภาษาการโปรแกรม
  2. คือการตรวจสอบว่ามีการใช้งานโอเปอแรนด์สอดคล้องกับโอเปอเรเตอร์หรือไม่ เช่น หากเครื่องหมาย + คือตัวดำเนินการบวก และภาษาการโปรแกรมกำหนดให้ใช้กับตัวแปรประเภทอินเทเจอร์เท่านั้น การทำ Type checking ก็จะเป็นการตรวจหาว่าตัวแปรที่เกี่ยวข้องกับ + นั้นเป็นตัวแปรประเภทอินเทเจอร์หรือไม่ หากไม่ใช่ก็จะรายงานข้อผิดพลาดให้แก้ไขโปรแกรม

ปกติข้อผิดพลาดบางส่วนจะถูกขจัดออกไปในกระบวนการคอมไพล์ แต่ข้อผิดพลาดบางอย่างอาจไม่สามารถตรวจได้ในเวลาคอมไพล์ แต่สามารถตรวจจับได้ในเวลารัน เช่น เช่น Memory พื้นที่ใน Hardisk หรือการเรียกใช้ทรัพยากรอื่นๆที่มีความเกี่ยวข้องกับการทำงานของโปรแกรม เช่น ไฟล์ และฐานข้อมูลเป็นต้น ในภาษาการโปรแกรมบางภาษามีคำสั่งช่วยจัดการข้อผิดพลาด โดยให้นักเขียนโปรแกรมกำหนดการตรวจจับเอง เพื่อเขียนคำสั่งตอบสนองต่อเหตุการณ์ข้อผิดพลาดต่างๆที่อาจเกิดขึ้นได้ในเวลารัน เพื่อเลี่ยงหรือลดความเสียหายจากการทำงานของโปรแกรมลง คำสั่งและการใช้งาน ในภาษาการโปรแกรมบางภาษาที่มีพอยน์เตอร์และการนำไปใช้ที่หลากหลายของตัวดำเนินการและพอยน์เตอร์อาจนำมาซึ่งข้อผิดพลาดในการเขียนโปรแกรมได้ง่าย ภาษาที่มีคุณลักษณะที่เชื่อถือได้ควรมี ที่ไม่หลากหลายจนเกินไป เพื่อลด ที่อาจเกิดขึ้น ตัวอย่างการเกิดข้อผิดพลาดที่เกิดขึ้นในเวลารัน ในภาษา Java
