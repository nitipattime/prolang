กระบวนทัศน์อื่นๆ

นักวิชาการหรือหนังสือบางเล่มอาจมีการกำหนดหรือนิยามกระบวนทัศน์ในรูปแบบอื่นๆ นอกเหนือจากสี่รูปแบบที่กล่าวมาเช่น คือ การเขียนโปรแกรมที่มีการออกแบบการทำงานซึ่งประกอบด้วย กลุ่ม (เช่น หรือ และกลุ่มของ การเขียนโปรแกรมเชิงโครงสร้าง มีความสัมพันธ์กับการแบ่งส่วนต่างๆของโปรแกรมตาม โดยตรง การโปรแกรมในลักษณะ Top-Down-Stepwise Refinement เป็นการแบ่งปัญหาออกเป็นส่วนย่อย ๆ ซึ่งแต่ละส่วนนั้นมีการทำงานเฉพาะเจาะจง ทำการเขียนโปรแกรมในแต่ละส่วนย่อยนั้นให้สมบูรณ์ แล้วนำแต่ละส่วนย่อยนั้นประกอบกันเป็นโปรแกรมซึ่งสามารถแก้ปัญหาทั้งหมดได้ การเขียนโปรแกรมในลักษณะนี้เป็นหลักการที่ใช้ในการเขียนโปรแกรมเชิงโครงสร้าง

คือ การเขียนโปรแกรมที่มีการออกแบบการทำงานซึ่งประกอบด้วยขั้นตอน และกลุ่มของการเรียกใช้ขั้นตอน ขั้นตอนคือโปรแกรมย่อย มีข้อแตกต่างจากการโปรแกรมเชิงโครงสร้างตรงที่หลักการนี้ไม่ได้เจาะจงว่าการเขียนโปรแกรมจะต้องพิจารณาการเขียนโปรแกรมในลักษณะ Top-Down-Stepwise Refinement เสมอไป

คือ การโปรแกรมที่จัดการการประมวลผลของโปรเซสหลายๆโปรเซสที่ทำงานพร้อม ๆ กันในเวลาเดียวกัน ซึ่งโปรเซสเหล่านั้นอาจมีการรบกวนการทำงานระหว่างกันและอาจมีการใช้ทรัพยากรร่วมกัน ตัวอย่างการโปรแกรมแบบ Concurrent programming ได้แก่ แต่ละเธรดจะรันโปรแกรมเป็นอิสระจากกัน (โปรแกรมอาจจะเหมือนหรือต่างกันก็ได้) และเธรดอาจมีการสื่อสารกันเพื่อจัดการการประสานงานในการเข้าถึงทรัพยากรที่ใช้งานร่วมกันเพื่อให้มีเพียงหนึ่งเธรดเท่านั้นที่สามารถเข้าถึงทรัพยากรได้ในเวลาหนึ่งๆ ภาษาการโปรแกรมสมัยใหม่ล้วนสนับสนุนการโปรแกรมในลักษณะ Concurrent programming เช่น ภาษา Java ภาษา C# และภาษา C++
